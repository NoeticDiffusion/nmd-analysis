# Remaining major concern

## The event/response-rate confound is now acknowledged, but still unresolved

The manuscript now correctly identifies event/response-rate structure as the main unresolved non-geometric alternative explanation. It explicitly states that GG is long and self-paced, while CC is fast and back-to-back, and that epoch-level event/response density could inflate Jacobian deformation estimates through a temporal-structure mechanism unrelated to task content. 

This is exactly the right limitation.

But from a reviewer perspective, it remains the most serious weakness. The primary result is robust to many confounds, but not to this one.

The current safe claim is:

> **MNJ differentiates the GG task block from the CC task block.**

The unsafe claim would be:

> **MNJ isolates gambling-specific or uncertainty-specific neural dynamics.**

The manuscript mostly avoids the unsafe claim, but some Discussion wording still leans toward a process interpretation.

### Recommended fix

Add one sentence to the Abstract or final paragraph:

> “Because GG and CC also differ in event/response density, the present result should be interpreted as a task-context difference rather than a process-pure gambling-specific mechanism.”

You already say this in Limitations and Conclusion, but adding it earlier would prevent reviewer backlash.

### Optional analysis if feasible

A crude event-density sensitivity would strengthen the paper:

* use sync/button-pulse density as a proxy,
* compare time-matched sub-blocks,
* downsample CC epochs to approximate GG pacing,
* exclude epochs close to response events,
* or include pulse proximity/density as an epoch-level covariate.

If this is hard, the current limitation is acceptable, but the paper should be submitted as **task-context geometry**, not mechanism isolation.

## Follow-up (2026-07-02): optional event-density sensitivity analysis completed

The "optional analysis if feasible" flagged above (lines 112–122) — a crude event-density sensitivity check using sync/button-pulse density — was subsequently implemented as `11_event_density_sensitivity.py`, joining the raw BIDS `events.tsv` Sync(1) trigger-pulse timestamps (dataset-native, no NMD re-ingest required) to the existing H5 epoch-level Jacobian windows. Four of the five suggested approaches were implemented (sync-pulse density as a proxy; a confound-partial correlation; an epoch-level density covariate; and a density-matched low-event-density-epochs-only re-test; time-matched sub-blocks / CC-downsampling were not additionally implemented since the density-matched test already addresses the same underlying question). Findings:

1. **Session-level event rate reverses the naive assumption.** Per-second Sync(1) pulse rate is *higher* in CC (median 0.453 Hz) than GG (median 0.357 Hz) — GG has far more total pulses per session (~1,285 vs. ~470) but a proportionally much longer session (~60 min vs. ~17 min median), so its per-second rate is actually lower. A purely event-density account of GG$>$CC Frobenius would predict the opposite direction of effect from what is observed.
2. **Subject-level confound-partial test is null.** Spearman(GG−CC event-rate diff, GG−CC Frobenius diff): $r_s = -0.05$, $p = 0.77$, $N = 44$ — added as a 7th row to the combined confound table (previously 6 rows).
3. **Epoch-level covariate is real but tiny.** Within-task Spearman(epoch event count, epoch Frobenius): $r_s \approx 0.03$ in both GG ($p = 2\times10^{-9}$, $n=39{,}985$) and CC ($p = 6.8\times10^{-4}$, $n=11{,}363$) — statistically significant given the large $n$, but $r^2 \approx 0.001$.
4. **Density-matched sensitivity confirms robustness.** Restricting the GG-vs-CC contrast to each subject's below-median-event-density epochs in both tasks leaves the effect essentially unchanged ($d = 1.70$ vs. $d = 1.62$ full-sample).

This substantially narrows, though does not fully close, the event/response-rate alternative (residual epoch-level coupling and the dataset's lack of stimulus/response pulse-type labels remain caveats).
